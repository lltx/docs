## Embedding模型是什么 

+ <font style="color:rgb(51, 51, 51);">定义：将文本数据转换为数值向量。</font>
+ <font style="color:rgb(51, 51, 51);">常用方法：Word2Vec、GloVe、FastText、BERT等。</font>
+ <font style="color:rgb(51, 51, 51);">优点：捕捉词汇之间的复杂关系。</font>
+ <font style="color:rgb(51, 51, 51);">应用：广泛用于文本分类、情感分析等NLP任务。</font>
+ <font style="color:rgb(51, 51, 51);">动态嵌入：如BERT，生成上下文相关的词嵌入。</font>

## LangChain4j与Qdrant的使用教程
本教程将涵盖如何安装Qdrant、如何使用LangChain4j与Qdrant进行向量查询和过滤查询，并演示相关代码。

#### 1. 安装Qdrant
首先，确保已经安装了Qdrant。Qdrant是一款高性能的向量数据库，用于存储嵌入并进行快速的向量搜索。以下是在本地机器上安装Qdrant的步骤：

##### 1.1 Docker安装Qdrant
Qdrant推荐使用Docker进行安装，这样可以避免复杂的环境配置：

```bash
docker pull qdrant/qdrant
docker run -p 6333:6333 -p 6334:6334 qdrant/qdrant
```

Qdrant默认使用两个端口：

+ **6333**：用于HTTP API
+ **6334**：用于gRPC API

##### 1.2 配置客户端
在Java项目中，我们可以使用gRPC API与Qdrant进行通信。使用`langchain4j`时，可以通过gRPC端口连接Qdrant，如上面的代码所示。

#### 2. 集成LangChain4j与Qdrant
下面是如何使用LangChain4j与Qdrant进行向量搜索的详细说明。

##### 2.1 配置Qdrant Embedding Store
首先，需要配置Qdrant的向量存储 (`EmbeddingStore`)，并指定Qdrant的主机地址和gRPC端口。以下是配置代码：

```java
EmbeddingStore<TextSegment> embeddingStore =
        QdrantEmbeddingStore.builder()
                .host("127.0.0.1")  // Qdrant服务器地址
                .port(grpcPort)      // Qdrant gRPC端口
                .collectionName(collectionName)  // Qdrant集合名称
                .build();
```

`collectionName`是Qdrant中的向量集合，用于存储和管理你的向量数据。

##### 2.2 嵌入生成与存储
在这个例子中，我们使用OpenAI的嵌入模型生成文本的向量。可以通过`OpenAiEmbeddingModel`来实现：

```java
EmbeddingModel embeddingModel = OpenAiEmbeddingModel.builder()
        .apiKey(System.getenv("OPENAI_KEY"))  // 设置OpenAI的API密钥
        .modelName("text-embedding-3-small")  // 嵌入模型的名称
        .baseUrl("https://api.openai-hk.com/v1")  // OpenAI接口地址
        .logRequests(true)  // 是否记录请求日志
        .build();
```

通过以下代码将文本生成的向量添加到Qdrant的向量存储中：

```java
TextSegment segment1 = TextSegment.from("I've been to France twice.");
segment1.metadata().put("author", "Alice1");
Embedding embedding1 = embeddingModel.embed(segment1).content();
embeddingStore.add(embedding1, segment1);
```

##### 2.3 向量查询
在进行查询时，可以使用一个查询嵌入作为输入进行向量搜索：

```java
Embedding queryEmbedding = embeddingModel.embed("Did you ever travel abroad?").content();
EmbeddingSearchRequest embeddingSearchRequest = EmbeddingSearchRequest.builder()
        .queryEmbedding(queryEmbedding)
        .maxResults(1)  // 返回最匹配的结果
        .build();
EmbeddingSearchResult<TextSegment> searchResult = embeddingStore.search(embeddingSearchRequest);
```

##### 2.4 过滤查询
在一些场景下，你可能需要根据元数据进行过滤。例如，你可以通过元数据中的“author”字段来筛选出特定作者的内容。可以使用以下代码：

```java
EmbeddingSearchRequest embeddingSearchRequest = EmbeddingSearchRequest.builder()
        .queryEmbedding(queryEmbedding)
        .filter(metadataKey("author").isEqualTo("Alice"))  // 过滤条件
        .maxResults(1)
        .build();
```

通过`metadataKey`方法，可以指定查询的元数据字段，并进行值匹配。

以下是`Filter`接口的名称、功能和使用示例的表格：

| Filter名称 | 功能 | 使用示例 |
| --- | --- | --- |
| And | 同时满足多个条件 | `Filter.and(condition1, condition2)` |
| Or | 满足其中任意一个条件 | `Filter.or(condition1, condition2)` |
| Not | 不满足条件 | `Filter.not(condition)` |
| IsEqualTo | 等于 | `new IsEqualTo("field", "value")` |
| IsNotEqualTo | 不等于 | `new IsNotEqualTo("field", "value")` |
| IsGreaterThan | 大于 | `new IsGreaterThan("field", value)` |
| IsGreaterThanOrEqualTo | 大于等于 | `new IsGreaterThanOrEqualTo("field", value)` |
| IsLessThan | 小于 | `new IsLessThan("field", value)` |
| IsLessThanOrEqualTo | 小于等于 | `new IsLessThanOrEqualTo("field", value)` |
| IsIn | 在列表内 | `new IsIn("field", listOfValues)` |
| IsNotIn | 不在列表内 | `new IsNotIn("field", listOfValues)` |


### 创建过滤条件的方式：
1. new 方式：如上表中的方式，直接通过构造器创建。
2. 链式创建：可以通过 `MetadataFilterBuilder` 进行链式创建，例如：

```java
MetadataFilterBuilder.metadataKey("field").isEqualTo("value")
```

这些`Filter`接口提供了丰富的条件过滤操作，能够实现复杂的逻辑组合，满足多种查询需求。

#### 3. 输出结果
最后，可以通过以下代码输出搜索结果：

```java
List<EmbeddingMatch<TextSegment>> relevant = searchResult.matches();
EmbeddingMatch<TextSegment> embeddingMatch = relevant.get(0);

System.out.println(embeddingMatch.score());  // 输出匹配分数
System.out.println(embeddingMatch.embedded().text());  // 输出匹配文本
```

本教程展示了如何使用LangChain4j与Qdrant进行向量查询、嵌入生成以及元数据过滤查询。通过这些步骤，你可以快速搭建基于Qdrant的高效向量搜索系统。
