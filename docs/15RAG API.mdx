## RAG API 文档

LangChain4j 提供了一套丰富的API，简化了构建自定义RAG（检索增强生成）管道的过程，支持从简单到复杂的多种管道实现。在本节中，我们将介绍主要的领域类和API。

### Document（文档）
`Document`类表示一个完整的文档，例如单个PDF文件或网页。目前，`Document`只能表示文本信息，但未来更新将支持图像和表格。

<details>
<summary>常用方法</summary>

- `Document.text()` 返回`Document`的文本内容
- `Document.metadata()` 返回`Document`的元数据（见下文）
- `Document.toTextSegment()` 将`Document`转换为`TextSegment`（见下文）
- `Document.from(String, Metadata)` 从文本和元数据创建`Document`
- `Document.from(String)` 从文本创建不带元数据的`Document`
</details>

### Metadata（元数据）
每个`Document`包含`Metadata`，存储文档的元信息，如名称、来源、最后更新时间、所有者等相关细节。元数据以键值对的形式存储，键为`String`类型，值可以是`String`、`Integer`、`Long`、`Float`、`Double`。

`Metadata`的用途包括：
- 当将文档内容包含在提示词中时，元数据条目也可以包含在内，为LLM提供额外信息以帮助理解内容。
- 在搜索相关内容时，可以按元数据条目进行过滤，例如限制搜索仅返回特定所有者的文档。
- 当文档来源更新时，可以根据元数据定位并更新`EmbeddingStore`中的对应文档以保持同步。

<details>
<summary>常用方法</summary>

- `Metadata.from(Map)` 从`Map`创建`Metadata`
- `Metadata.put(String key, String value)` 添加元数据条目
- `Metadata.getString(String key)` / `getInteger(String key)` 获取指定类型的元数据值
- `Metadata.containsKey(String key)` 检查是否包含指定键
- `Metadata.remove(String key)` 移除指定键的元数据条目
- `Metadata.copy()` 复制元数据
- `Metadata.toMap()` 将元数据转换为`Map`
</details>

### Document Loader（文档加载器）
可以通过`String`创建`Document`，但更简单的方法是使用库中提供的文档加载器：
- `FileSystemDocumentLoader` 从文件系统加载文档
- `UrlDocumentLoader` 从URL加载文档
- `AmazonS3DocumentLoader` 从Amazon S3加载文档
- `AzureBlobStorageDocumentLoader` 从Azure Blob存储加载文档
- `GitHubDocumentLoader` 从GitHub仓库加载文档
- `TencentCosDocumentLoader` 从腾讯云COS加载文档

### Document Parser（文档解析器）
`Document`可以表示多种文件格式，如PDF、DOC、TXT等。为了解析这些格式，库中提供了`DocumentParser`接口及其实现：
- `TextDocumentParser` 解析纯文本文件
- `ApachePdfBoxDocumentParser` 解析PDF文件
- `ApachePoiDocumentParser` 解析MS Office文件格式
- `ApacheTikaDocumentParser` 自动检测并解析几乎所有文件格式

文档加载示例：
```java
// 加载单个文档
Document document = FileSystemDocumentLoader.loadDocument("/home/langchain4j/file.txt", new TextDocumentParser());

// 加载目录下的所有文档
List<Document> documents = FileSystemDocumentLoader.loadDocuments("/home/langchain4j", new TextDocumentParser());
```

### Document Transformer（文档转换器）
`DocumentTransformer`用于对文档执行各种转换，例如：
- 清理：删除文档文本中的噪音，节省token并减少干扰。
- 过滤：完全排除某些不相关的文档。
- 增强：向文档添加额外的信息以提高搜索结果质量。
- 总结：将文档总结，存储在`Metadata`中，供后续使用。

当前库中提供了`HtmlToTextDocumentTransformer`，可以从HTML中提取文本内容和元数据。建议根据自身需求实现自定义的`DocumentTransformer`。

### Text Segment（文本片段）
加载文档后，接下来可以将其拆分为更小的片段。LangChain4j中使用`TextSegment`类表示文档的一个片段，专用于文本信息。

<details>
<summary>是否需要拆分？</summary>

拆分文档有几个原因：
- LLM的上下文窗口有限，无法容纳整个知识库。
- 提供的上下文越多，LLM处理和响应所需的时间越长。
- 提供的上下文越多，成本越高。
- 提供的无关信息可能会干扰LLM并增加幻觉的可能性。
- 提供的上下文越多，越难解释LLM的响应依据。

通常有两种拆分方法：
1. 将每个文档视为不可分割的整体，直接检索最相关的文档并注入到提示中。这种方法适合不允许丢失细节的场景，但会消耗更多token。
2. 将文档拆分为更小的片段，如章节或段落，在检索时只返回最相关的片段。这种方法减少了token消耗，但可能会丢失一些上下文。

</details>

<details>
<summary>常用方法</summary>

- `TextSegment.text()` 返回`TextSegment`的文本
- `TextSegment.metadata()` 返回`TextSegment`的元数据
- `TextSegment.from(String, Metadata)` 从文本和元数据创建`TextSegment`
- `TextSegment.from(String)` 从文本创建不带元数据的`TextSegment`
</details>

### Document Splitter（文档拆分器）
LangChain4j 提供了多种`DocumentSplitter`实现：
- `DocumentByParagraphSplitter` 按段落拆分
- `DocumentBySentenceSplitter` 按句子拆分
- `DocumentByWordSplitter` 按单词拆分
- `DocumentByCharacterSplitter` 按字符拆分
- `DocumentByRegexSplitter` 按正则表达式拆分

拆分步骤：
1. 实例化`DocumentSplitter`并指定`TextSegment`的大小和重叠字符数。
2. 调用`split(Document)`或`splitAll(List<Document>)`方法。
3. `DocumentSplitter`将文档拆分成小片段，并将其组合为`TextSegment`。

### Embedding（嵌入）
`Embedding`类封装了一个数值向量，表示嵌入内容的语义含义，通常为文本。

<details>
<summary>常用方法</summary>

- `Embedding.dimension()` 返回嵌入向量的维度
- `CosineSimilarity.between(Embedding, Embedding)` 计算两个嵌入向量的余弦相似度
- `Embedding.normalize()` 对嵌入向量进行归一化
</details>

### Embedding Model（嵌入模型）
`EmbeddingModel`接口表示一种模型，将文本转换为`Embedding`。

<details>
<summary>常用方法</summary>

- `EmbeddingModel.embed(String)` 嵌入指定文本
- `EmbeddingModel.embed(TextSegment)` 嵌入指定`TextSegment`
- `EmbeddingModel.embedAll(List<TextSegment>)` 嵌入多个`TextSegment`
- `EmbeddingModel.dimension()` 返回嵌入向量的维度
</details>

### Embedding Store（嵌入存储）
`EmbeddingStore`接口表示一个嵌入存储库（向量数据库），用于存储和高效搜索相似的嵌入。

<details>
<summary>常用方法</summary>

- `EmbeddingStore.add(Embedding)` 添加嵌入并返回随机ID
- `EmbeddingStore.addAll(List<Embedding>)` 添加多个嵌入并返回随机ID列表
- `EmbeddingStore.search(EmbeddingSearchRequest)` 搜索最相似的嵌入
- `EmbeddingStore.remove(String id)` 删除指定ID的嵌入
</details>

### Embedding Store Ingestor（嵌入存储摄取器）
`EmbeddingStoreIngestor`负责将文档嵌入并存储到`EmbeddingStore`中。

示例：
```java
EmbeddingStoreIngestor ingestor = EmbeddingStoreIngestor.builder()
    .embeddingModel(embeddingModel)
    .embeddingStore(embeddingStore)
    .build();

ingestor.ingest(document1);
```

可以通过指定`DocumentTransformer`和`DocumentSplitter`，在嵌入前对文档进行转换和拆分。
