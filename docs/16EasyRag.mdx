# RAG（检索增强生成）

LLM 的知识有限于其训练数据。如果你想让 LLM 获得特定领域的知识或专有数据，有以下几种方法：
- 使用 RAG（我们将在本节中详细介绍）
- 使用你的数据对 LLM 进行微调
- [结合使用 RAG 和微调](https://gorilla.cs.berkeley.edu/blogs/9_raft.html)

## 什么是 RAG？
简单来说，RAG 是在将提示（prompt）发送给 LLM 之前，从你的数据中找到并注入相关信息的方法。通过这种方式，LLM 可以获取相关信息并基于此回复，从而降低幻觉的概率。

相关信息可以通过多种[信息检索](https://en.wikipedia.org/wiki/Information_retrieval)方法找到，最常见的有：
- 全文搜索（关键词搜索）。这类方法使用如 TF-IDF 和 BM25 的技术，通过将查询中的关键词与文档数据库匹配来搜索文档，并根据这些关键词在每个文档中的频率和相关性进行排名。
- 向量搜索（语义搜索）。文本文档被转换为数值向量，利用嵌入模型通过余弦相似度或其他相似性/距离度量来查找和排名文档，捕捉更深层的语义。
- 混合搜索。结合全文搜索和向量搜索的方法，通常会提高搜索的有效性。

本文重点介绍向量搜索。全文搜索和混合搜索目前仅在 Azure AI Search 集成中支持，更多功能将在未来扩展到 RAG 工具包中。

## RAG 流程
RAG 过程分为两个阶段：索引（indexing）和检索（retrieval）。LangChain4j 提供了这两个阶段的工具。

### 索引阶段
在索引阶段，文档经过预处理以便在检索阶段进行高效搜索。对于向量搜索，通常包括清理文档、添加额外数据和元数据、将文档分割为小段（分块），将这些段落嵌入为向量，并存储在嵌入存储（向量数据库）中。

索引通常是离线进行的，可以通过定期任务（如每周重建索引）完成。

<img src='https://minio.pigx.top/oss/202410/1728447018.png' alt='1728447018'/>

### 检索阶段

检索阶段通常是在线进行的，当用户提交问题时，从索引的文档中寻找相关信息。对于向量搜索来说，这通常包括将用户的查询嵌入为向量，并在嵌入存储中执行相似性搜索，找到相关的段落并将它们注入提示中，再发送给 LLM。

## Easy RAG
LangChain4j 提供了“Easy RAG”功能，旨在让你快速上手 RAG，无需学习嵌入、选择向量存储或如何解析和分割文档等复杂步骤。只需指向你的文档，LangChain4j 将为你处理大部分细节。

1. 导入 `langchain4j-easy-rag` 依赖：
```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-easy-rag</artifactId>
    <version>0.35.0</version>
</dependency>
```

2. 加载文档：
```java
List<Document> documents = FileSystemDocumentLoader.loadDocuments("/home/langchain4j/documentation");
```

3. 预处理并存储文档：
```java
InMemoryEmbeddingStore<TextSegment> embeddingStore = new InMemoryEmbeddingStore<>();
EmbeddingStoreIngestor.ingest(documents, embeddingStore);
```

4. 创建一个 AI 服务接口：
```java
interface Assistant {
    String chat(String userMessage);
}

ChatLanguageModel chatModel = OpenAiChatModel.builder()
    .apiKey(System.getenv("OPENAI_API_KEY"))
    .modelName(GPT_4_O_MINI)
    .build();

Assistant assistant = AiServices.builder(Assistant.class)
    .chatLanguageModel(chatModel)
    .chatMemory(MessageWindowChatMemory.withMaxMessages(10))
    .contentRetriever(EmbeddingStoreContentRetriever.from(embeddingStore))
    .build();
```

5. 开始对话：
```java
String answer = assistant.chat("How to do Easy RAG with LangChain4j?");
```
