title: 提示词工程
---

在本文中,我们将探讨如何利用LangChain4j框架构建一个专业的英语翻译助手。这个助手将专注于中文到英语的翻译,对其他任务则会礼貌地拒绝。我们将深入了解角色设定和提示词模板的使用,这是实现这个功能的两个关键要素。

## 角色设定:塑造AI助手的专业身份
角色设定是指导大语言模型(LLM)行为的重要手段。通过明确定义AI助手的身份和能力范围,我们可以使其更专注于特定领域的任务。在LangChain4j中,我们主要利用`SystemMessage`来实现这一点。

`SystemMessage`具有高优先级,能有效地指导模型的整体行为。例如,我们可以这样定义一个专业翻译官的角色:

```java
@SystemMessage("你是一位专业的翻译官,专门负责中文到英语的翻译。对于其他语言或任务,请礼貜地表示无法处理。")
```

## 提示词模板:精确控制输入输出
LangChain4j提供了多种方式来使用提示词模板,让我们能够灵活地构造输入并控制输出。以下是几种常用的方法:

1. 使用 `@UserMessage` 和 `@V` 注解:

```java
@UserMessage("请将以下文本翻译成{{language}}: {{text}}")
String translate(@V("text") String text, @V("language") String language);
```

2. 使用 `@StructuredPrompt` 定义结构化提示:

```java
@Getter
@AllArgsConstructor
@StructuredPrompt("将{{sourceText}}翻译成{{targetLanguage}}")
class TranslationPrompt {
    private String sourceText;
    private String targetLanguage;
}
```

3. 使用 `PromptTemplate`:

```java
PromptTemplate template = PromptTemplate.from("将'{{it}}'翻译成英语。");
Prompt prompt = template.apply("你好");
System.out.println(prompt.text()); // 输出: 将'你好'翻译成英语。
```

## 实现专业英语翻译助手
现在,让我们来实现这个专业的英语翻译助手。

首先,定义 `Translator` 接口:

```java
package com.pig4cloud.ai.service;

import dev.langchain4j.service.SystemMessage;
import dev.langchain4j.service.UserMessage;
import dev.langchain4j.service.V;
import dev.langchain4j.service.spring.AiService;

@AiService
public interface Translator {

    @SystemMessage("你是一位专业的翻译官,只负责中文到英语的翻译。对于其他语言或任务,请回复'抱歉,我只能翻译中文到英语。'")
    @UserMessage("请将以下文本翻译成{{language}}: {{text}}")
    String translate(@V("text") String text, @V("language") String language);
}
```

然后,创建一个控制器来处理翻译请求:

```java
package com.pig4cloud.ai.controller;

import com.pig4cloud.ai.service.Translator;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;

/**
 * 翻译控制器
 *
 * @author lengleng
 */
@RestController
public class TranslatorController {

    private final Translator translator;

    public TranslatorController(Translator translator) {
        this.translator = translator;
    }

    @GetMapping("/translate")
    public String translate(@RequestParam("text") String text, @RequestParam("language") String language) {
        return translator.translate(text, language);
    }
}
```

## 测试与效果
使用这个翻译助手,我们可以得到以下效果:

1. 中文到英语的翻译:  
输入: "你好,世界!"  
输出: "Hello, world!"
2. 尝试其他语言:  
输入: "将'你好'翻译成日语"  
输出: "抱歉,我只能翻译中文到英语。"

## 总结
通过LangChain4j框架,我们成功实现了一个专注于中英翻译的AI助手。关键点在于:

1. 使用`SystemMessage`明确定义助手的角色和能力范围。
2. 利用提示词模板(`@UserMessage`, `@V`)精确控制输入和期望的输出格式。
3. 在接口中组合使用这些注解,轻松实现复杂的AI交互逻辑。

这个例子展示了LangChain4j强大的灵活性和易用性,为开发者提供了构建专业化AI应用的有力工具。通过合理设置角色和精心设计提示词,我们可以让AI助手更好地服务于特定的业务需求。
