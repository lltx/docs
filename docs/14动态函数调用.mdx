LangChain4j系列：LangChain4j 接入GraalVm
 实现动态函数调用

#### 引言

在现代软件开发中，代码执行引擎的应用场景越来越广泛。LangChain4j框架通过集成多种代码执行引擎，为开发者提供了强大的工具支持。本文将详细介绍LangChain4j如何接入GraalVm，实现动态函数调用，并探讨其在实际业务场景中的应用。

#### 代码执行引擎的使用场景


#### LangChain4j框架支持的执行引擎

LangChain4j目前支持两种主要的代码执行引擎：

1. **GraalVM**：
    - **Polyglot**：允许在同一个应用程序中无缝使用多种编程语言。
    - **Truffle**：用于构建语言引擎的框架，支持轻松添加新语言支持。
2. **Judge0**：
    - 一个开源的代码执行引擎，支持多种编程语言，如C、C++、Java、Python、Ruby等。
    - 号称“世界上最先进的开源在线代码执行系统”。

#### 使用示例

##### 依赖配置
```xml
<dependency>
<groupId>dev.langchain4j</groupId>
<artifactId>langchain4j-code-execution-engine-graalvm-polyglot</artifactId>
</dependency>
```

##### 配置类
```java
ChatAssistant assistant = AiServices.builder(ChatAssistant.class)
        .chatLanguageModel(chatLanguageModel)
        .tools(new GraalVmJavaScriptExecutionTool())
        .build();
```

##### Ai Service 定义

```java
public interface Assistant {
    String chat(String message);
}
```

##### 测试

```java
String chat = chatAssistant.chat("What is the square root of 485906798473894056 in scientific notation?");
System.out.println(chat);
```
